# Metrics and Instrumentation Spec

## Purpose

Define how HOTMESS measures:
- truth integrity
- trust health
- readability
- saturation
- launch readiness
- abuse resistance

Metrics are operational controls.

---

# Priority Order

1. Safety metrics
2. Truth metrics
3. Trust metrics
4. Readability metrics
5. Density metrics
6. Engagement metrics
7. Monetization metrics

---

# Core Rules

Metrics must be:
- bounded
- actionable
- segmentable
- alertable
- time-aware

Metrics must not be vanity analytics.

---

# Safety Metrics

Track:
- harassment rate
- unsafe visibility rate
- stalking-risk suppressions
- moderator intervention rate
- repeat offender rate

---

# Truth Metrics

Track:
- stale leakage
- correction latency
- contradiction rate
- false urgency rate
- venue accuracy rate
- event timing accuracy

---

# Trust Metrics

Track:
- trust distribution
- trust decay
- trust recovery
- verification success
- restricted account rate
- dispute frequency

---

# Readability Metrics

Track:
- cluster collapse rate
- clutter score
- label overlap
- abandonment during dense views
- visible item count

---

# Saturation Metrics

Track:
- source dominance
- paid visibility share
- boost concentration
- district congestion
- density volatility

---

# Ranking Quality Metrics

Track:
- useful route starts
- saves
- confirmations
- freshness alignment
- trusted relevance score

---

# Launch Readiness Metrics

Track:
- verified venue coverage
- moderation capacity
- stale leakage
- trust mix
- abuse detection latency
- district stability

---

# Alert Thresholds

Alert classes:
- green
- amber
- red
- critical

Every important metric must map to:
- threshold
- owner
- escalation path

---

# Core Invariants

- safety outranks monetization
- truth outranks engagement
- readability outranks density
- quiet states are valid
- stale leakage must stay low
- boost success is utility-based
