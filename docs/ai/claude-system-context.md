# Claude System Context — HOTMESS Globe

## Product Identity

HOTMESS is a realtime queer nightlife intelligence and coordination platform.

It is not:
- a social network
- a dating app
- an infinite feed
- an engagement maximizer
- a generic nightlife directory

It is:
- realtime decision-support infrastructure
- governed visibility architecture
- trust-weighted nightlife intelligence
- density-aware coordination software

## Core Product Question

What is happening right now, and where should I go?

## Core System Priorities

1. Safety
2. Truth
3. Trust
4. Freshness
5. Readability
6. Relevance
7. Monetization

## Immutable Invariants

- Safety outranks everything
- Truth outranks trust
- Trust outranks payment
- Freshness outranks popularity
- Momentum outranks dead volume
- Readability outranks completeness
- Quiet states are valid
- Boosts are bounded
- Stale data must not masquerade as live data
- Exact user tracking is forbidden by default

## Platform Philosophy

HOTMESS is a managed visibility economy.

Core primitives:
- rank
- scarcity
- trust
- amplification
- suppression
- decay

NOT:
- endless engagement
- creator favoritism
- artificial urgency
- synthetic activity

## Operational Doctrine

When uncertain:
- preserve readability
- preserve trust
- reduce amplification
- reduce certainty
- collapse density earlier
- require corroboration
- favor truthful quiet over synthetic activity

## Rendering Doctrine

The map must remain readable.

High density should:
- cluster
- compress
- prioritize representative leaders
- suppress repetition

Never allow:
- unreadable clutter
- sponsor dominance
- stale visibility persistence

## Trust Doctrine

Trust is operational permission.

Trust affects:
- ranking
- amplification
- decay
- persistence
- throttling
- moderation thresholds

Trust is:
- earned
- contextual
- reversible
- decayable

## Monetization Doctrine

Payment may improve visibility only within bounded policy limits.

Payment must never:
- override trust
- override safety
- create fake urgency
- dominate districts permanently
- break readability

## Quiet-State Doctrine

Quiet states are valid.

Do not:
- fake activity
- manufacture urgency
- inflate density artificially

Truthful low-density states are preferable to synthetic engagement.

## Engineering Rule

If a rule cannot be:
- measured
- enforced
- audited
- thresholded
- rate-limited
- state-transitioned

then it is not production-ready.
