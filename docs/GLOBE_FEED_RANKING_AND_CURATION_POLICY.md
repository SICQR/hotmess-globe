# Globe Feed Ranking And Curation Policy

Purpose: define how ranking, curation, prioritisation, suppression, visibility weighting, and content ordering behave across HOTMESS Globe feed systems.

This document governs:

- feed ranking;
- contextual prioritisation;
- curation systems;
- visibility decay;
- suppression systems;
- safety weighting;
- anti-spam governance;
- density handling;
- sponsored placement;
- realtime ordering.

The Globe should feel:

```txt
curated, calm, contextual, and culturally alive
```

not:

```txt
manipulative, addictive, popularity-driven, or algorithmically chaotic
```

---

# Core philosophy

Ranking exists to:

- reduce noise;
- improve orientation;
- surface relevance;
- surface atmosphere;
- preserve emotional calm;
- support safety;
- support contextual discovery.

Ranking does NOT exist to:

- maximise engagement addiction;
- maximise session time;
- maximise outrage;
- maximise vanity metrics;
- maximise social competition.

---

# Non-negotiable anti-gamification policy

Feed systems must NEVER:

- publicly rank users;
- reward thirst traps algorithmically;
- amplify harassment engagement;
- expose popularity ladders;
- incentivise spam posting;
- manufacture virality loops.

Forbidden:

```txt
Most popular nearby
```

```txt
Trending profiles
```

```txt
Top beacon tonight
```

```txt
People everyone wants right now
```

---

# Ranking philosophy

The Globe is:

```txt
an atmosphere engine
```

not:

```txt
a clout machine
```

Feeds should feel:

- finite;
- breathable;
- local;
- emotionally calm;
- spatially contextual.

# Freshness weighting

Freshness matters.

But freshness must:

- decay smoothly;
- avoid jitter;
- avoid aggressive reshuffling.

Good:

```txt
recently active
```

Bad:

```txt
LIVE RIGHT NOW!!!
```

without proof.

---

# Contextual ranking

Ranking should adapt to:

- district;
- time of day;
- local activity;
- saved context;
- accessibility needs;
- reduced stimulation mode;
- event proximity;
- user intent.

Example:

A user in:

```txt
reduced stimulation mode
```

should NOT receive:

- aggressive nightlife density;
- rapid feed reshuffling;
- high-frequency pulse surfaces.

---

# Spatial relevance

Spatial ranking should use:

- approximate proximity bands;
- district grouping;
- neighbourhood context;
- venue adjacency.

The system must NEVER:

- expose exact distance precision;
- expose exact movement tracking;
- expose route behaviour.

Preferred:

```txt
around Soho
```

Forbidden:

```txt
31 metres away
```

---

# Feed curation model

Feeds should combine:

- algorithmic relevance;
- editorial atmosphere;
- realtime freshness;
- safety weighting;
- local context.

The system should feel:

```txt
guided rather than manipulated
```

---

# Editorial curation

Editorial systems may:

- highlight major cultural moments;
- highlight verified events;
- highlight important care resources;
- highlight HOTMESS RADIO moments;
- highlight accessibility-safe venues.

Editorial systems must NOT:

- fabricate hype;
- artificially manufacture scarcity;
- simulate fake popularity.

---

# Density handling

Dense districts require aggregation.

Instead of:

```txt
824 overlapping beacons
```

Use:

```txt
Warm district activity tonight
```

or:

```txt
Several active social signals nearby
```

Ranking should prioritise:

- readability;
- calmness;
- emotional clarity.

---

# Beacon ranking

Beacon visibility may depend on:

- freshness;
- moderation state;
- spam weighting;
- local context;
- expiry window;
- user intent.

Beacon visibility must NEVER depend on:

- beauty hierarchy;
- social popularity;
- public Boo counts.

---

# Chill ranking

Chill signals should remain:

- lightweight;
- ambient;
- non-performative.

Avoid:

- competitive ordering;
- desirability ladders;
- engagement optimisation.

Good:

```txt
Nearby Chill signals
```

Bad:

```txt
Most wanted nearby
```

---

# Event ranking

Event ranking may use:

- proximity;
- freshness;
- ticket validity;
- saved preferences;
- accessibility match;
- district relevance.

Event ranking must NOT:

- favour wealth signalling;
- favour exclusivity theatre artificially.

---

# Care ranking

Care systems are protected infrastructure.

Care visibility should prioritise:

- trust;
- accessibility;
- calmness;
- safety;
- relevance.

Care systems must NEVER:

- become competitive;
- expose vulnerability publicly;
- use engagement ranking.

---

# Radio ranking

Radio ranking should feel:

- atmospheric;
- cultural;
- contextual.

Radio ranking may use:

- district energy;
- live status;
- saved listening;
- venue integrations.

Radio should NOT:

- interrupt aggressively;
- dominate local discovery.

---

# Market/preloved ranking

Market ranking should remain:

- secondary;
- sparse;
- contextual.

Market systems must NOT:

- dominate social feeds;
- use fake urgency;
- aggressively interrupt.

Forbidden:

```txt
Selling out now
```

unless verified.

---

# Discovery decay

All feed items should decay.

Decay factors:

- time;
- density;
- duplication;
- expiry;
- spam weighting;
- stale interaction.

Decay should:

- soften prominence;
- reduce interruption;
- reduce clutter.

---

# Feed suppression systems

Suppression engine may:

- de-rank spam;
- de-rank harassment;
- de-rank duplicate signals;
- suppress expired beacons;
- suppress manipulative behaviour.

Suppression should remain:

```txt
quiet and non-theatrical
```

---

# Realtime ranking behaviour

Realtime updates should:

- transition softly;
- avoid jumpiness;
- avoid constant reshuffling.

Avoid:

- slot-machine refresh mechanics;
- aggressive reorder flicker;
- addictive volatility.

---

# Feed interruption governance

The feed should NOT:

- hijack attention constantly;
- reorder dramatically;
- create urgency loops.

Preferred:

- stable ranking;
- gradual transitions;
- contextual updates.

---

# Personalisation policy

Personalisation should:

- reduce noise;
- improve relevance;
- support accessibility;
- support calm discovery.

Personalisation must NOT:

- trap users in feedback loops;
- intensify obsession;
- isolate users socially;
- reinforce unhealthy compulsions.

---

# Sponsored ranking policy

Sponsored placements:

- must be labelled;
- must remain sparse;
- must never imitate care systems;
- must never dominate district context.

Sponsored systems must NEVER:

- simulate local popularity;
- fake community activity;
- impersonate realtime signals.

---

# Ghosted chat feed policy

Ghosted interactions may surface:

- contextual reopen opportunities;
- mutual interaction relevance;
- calm social continuity.

Ghosted systems must NOT:

- expose rejection states;
- expose unread pressure;
- create attention anxiety.

---

# Emotional safety governance

Ranking systems must avoid:

- social shame;
- rejection amplification;
- desirability hierarchy;
- loneliness exploitation;
- scarcity manipulation;
- jealousy loops.

The feed should feel:

```txt
socially breathable rather than emotionally extractive
```

---

# Reduced stimulation mode

Reduced stimulation mode should:

- reduce density;
- reduce animations;
- reduce realtime churn;
- reduce nightlife intensity;
- simplify feed hierarchy.

The feed should feel:

```txt
quieter and slower
```

---

# Accessibility

Feed systems must support:

- reduced motion;
- reduced stimulation;
- screen readers;
- keyboard navigation;
- high contrast;
- low cognitive load.

Feed meaning must remain understandable without:

- animation;
- colour-only indicators;
- dense realtime transitions.

# Testing requirements

## Unit tests

Test:

- ranking fairness;
- decay timing;
- spam suppression;
- sponsored placement limits;
- reduced stimulation behaviour;
- accessibility-aware ranking;
- freshness weighting.

## Integration tests

Test:

- exact GPS never leaks;
- popularity factors remain excluded;
- dense districts aggregate correctly;
- care outranks market;
- realtime updates stay calm;
- suppression propagates correctly.

## E2E tests

Test:

- feeds feel contextual rather than addictive;
- discovery remains readable at high density;
- reduced stimulation materially changes ranking behaviour;
- spam beacons decay correctly;
- feed ordering remains stable under load;
- popularity never becomes dominant ranking behaviour.

---

# Acceptance criteria

The ranking system succeeds when:

- feeds feel curated rather than manipulative;
- users can orient quickly without overwhelm;
- discovery remains calm under realtime density;
- exact location never leaks;
- popularity does not dominate visibility;
- social interaction remains emotionally breathable;
- care remains protected infrastructure;
- realtime updates feel atmospheric rather than addictive;
- sponsored systems remain clearly secondary;
- the Globe feels culturally alive without becoming algorithmically toxic.
