# D48 §3.1 Amendment — Photo Gates the Anonymous Fallback

**Date:** 2026-06-03
**Ratified by:** Phil
**Status:** Locked

## What changed

The Ghosted card identity fallback chain is amended from:

```
profileName → display_name → 'Anonymous'
```

to:

```
profileName → display_name → username → (photo ? 'Member' : 'Anonymous')
```

## Why

Phil 2026-06-03 live observation: cards rendered "Anonymous" next to a fully visible
photo, green presence dot, proximity ("< 5 min away"), and an Active Now badge. The
profile sheet revealed the actual username (e.g. "Riikey2k16") on tap. The label
contradicted everything else on the card.

D48's privacy intent — concealment by default — is preserved for users without
photos (silhouette cards still anonymise). But once a face is visible, the
"Anonymous" label adds no privacy value, only friction. D44 mandates that every
user has a username, so the chain almost always lands on a real identifier
before the silhouette case fires.

## Doctrinal grammar

- **Photo + name:** show the name (no change).
- **Photo, no profileName/display_name:** show `username` (new behaviour — was
  "Anonymous" before this amendment).
- **No photo, has username:** show `username` (no change).
- **No photo, no username:** show `Anonymous` (legacy unfinished-profile case).
- **Photo, no name fields at all:** show `Member` (rare; defensive fallback).

## Surfaces affected

- `src/lib/ghostedUtils.ts` → `mapGhostedUser` (the card data shaper)
- Cascading: `GhostedCard.tsx`, the Ghosted grid, ProfilesGrid

## What did NOT change

- L2ProfileSheet identity reveal (still uses `username || profileName || display_name`).
- D48 §3.1 Option A matrix preamble — identity remains concealment-by-default at
  the spatial/proximity layer. This amendment only governs the *fallback string*,
  not the disclosure model.
- Silhouette cards still anonymise — the photo-less case is preserved.
