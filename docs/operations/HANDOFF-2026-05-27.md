## Workflow rules learned this session

### Pre-push gate (NEW — adopted 2026-05-27)

Sandbox bash timeout is **45s** and `npm run build` (vite) takes longer. The full build cannot complete inside a single bash call. So:

- **Local pre-push:** `npm run typecheck` only (~10s). Passes catch >90% of regressions.
- **Build gate:** Vercel preview deployment. If preview is green, code compiles.
- **Production safety:** if preview fails, revert the PR — don't try to debug inside the sandbox.

### Big-diff push strategy

A push of 2,000+ LOC of deletions (e.g. PR #549) can also exceed the 45s cap. Workarounds:
- Run the push in `nohup ... > /tmp/log 2>&1 < /dev/null &` and poll in the next bash call.
- Or just retry the synchronous push — second attempt often succeeds because objects are already negotiated.

### Production promote (CRITICAL — Vercel does NOT auto-promote main → production)

Production tracks the separate `production` branch. Merges to `main` only build **preview** deployments. To go live:

```python
# Commit-tree pattern (avoids merge conflicts on production branch)
1. Get main_tree_sha + production_head_sha
2. POST /repos/SICQR/hotmess-globe/git/commits
   { tree: main_tree_sha, parents: [production_head_sha], message: "promote: ..." }
3. PATCH refs/heads/release/sync-prod-XXX → new commit sha (force)
4. Open PR release/sync-prod-XXX → production
5. Merge PR via API (merge_method: "merge")
6. Vercel auto-builds + flips production alias
7. Verify: production tree sha == main tree sha
```

This pattern is used because `production` is branch-protected and direct ref patches return 422.

### Migration (PR-A first commit)
```sql
ALTER TABLE profiles ADD COLUMN beta_access_until timestamptz;

CREATE TABLE beta_invites (
  code text PRIMARY KEY,             -- 8-char random, e.g. PHIL-X7K2
  email text,                        -- optional pre-allocation
  uses_remaining int DEFAULT 1,
  expires_at timestamptz,            -- code itself expires (vs the access window)
  created_at timestamptz DEFAULT now(),
  created_by uuid REFERENCES auth.users(id),
  claimed_by uuid REFERENCES auth.users(id),
  claimed_at timestamptz,
  notes text                         -- "twitter dm 2026-05-27"
);
CREATE INDEX beta_invites_email_idx ON beta_invites(email);
CREATE INDEX beta_invites_claimed_idx ON beta_invites(claimed_by);

CREATE TABLE beta_feedback (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users(id),
  rating int CHECK (rating BETWEEN 1 AND 5),
  text text,
  screenshot_url text,
  path text,
  session_id text,
  device_id text,
  created_at timestamptz DEFAULT now()
);
CREATE INDEX beta_feedback_user_idx ON beta_feedback(user_id);
CREATE INDEX beta_feedback_created_idx ON beta_feedback(created_at DESC);
```

### Premium gate update (one helper changes everywhere)
```ts
// src/lib/auth/premium.ts (or wherever isPremium lives)
export function isPremium(p?: { subscription_status?: string; beta_access_until?: string | null }) {
  if (!p) return false;
  if (p.subscription_status === 'active') return true;
  if (p.beta_access_until && new Date(p.beta_access_until) > new Date()) return true;
  return false;
}
```
