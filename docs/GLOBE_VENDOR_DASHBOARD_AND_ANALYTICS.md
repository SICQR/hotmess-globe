# Globe Vendor Dashboard And Analytics

Purpose: define the canonical vendor dashboard system for HOTMESS Globe partners, venues, creators, event operators, popup vendors, ticket operators, and sponsored beacon campaigns.

This document governs:

- vendor-facing tooling;
- beacon campaign management;
- analytics surfaces;
- trust and moderation visibility;
- subscription entitlements;
- privacy-safe reporting;
- realtime operational insight.

The dashboard must help vendors:

- understand local energy;
- manage campaigns;
- optimise discovery;
- coordinate events;
- improve accessibility;
- reduce spam;
- operate safely.

WITHOUT turning Globe into:

- adtech surveillance;
- casino dashboards;
- manipulative engagement systems;
- vanity analytics addiction;
- creepy location intelligence.

---

# Core philosophy

The vendor dashboard should feel:

```txt
clear, operational, cultural, realtime, trustworthy
```

NOT:

```txt
growth-hack panic software
```

The dashboard exists to:

- support nightlife infrastructure;
- support community organisers;
- support care-aware commerce;
- improve signal quality;
- reduce operational chaos.

NOT:

- maximise notifications;
- maximise addiction;
- expose user movement;
- encourage spam publishing.

# Accessibility requirements

Dashboard must support:

- reduced motion;
- keyboard navigation;
- screen readers;
- low stimulation mode;
- calm chart variants;
- colorblind-safe states.

Charts should:

- avoid visual overload;
- support list fallback;
- avoid flashing.

---

# Moderation requirements

Vendor tooling must support:

- disclosure enforcement;
- spam detection;
- scam prevention;
- anti-scalping rules;
- abuse escalation;
- policy warnings.

Repeated abuse should:

- reduce eligibility;
- reduce visibility;
- restrict boost access;
- suspend campaigns.

# Acceptance criteria

The dashboard succeeds when:

- vendors understand realtime energy clearly;
- operators can manage events calmly;
- creators improve discovery without spam;
- accessibility becomes operationally visible;
- monetisation remains restrained;
- analytics remain privacy-safe;
- district maps stay readable;
- moderation is enforceable;
- nightlife and care coexist safely;
- the dashboard feels like cultural infrastructure, not surveillance software.
