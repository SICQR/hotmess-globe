# HOTMESS Founder Source of Truth

This directory is the canonical doctrine layer for HOTMESS.

It exists to keep `hotmess-globe` aligned across product, copy, radio, commerce, care, moderation, and future AI-assisted builds.

## Relationship to the live product

`hotmess-globe` is the execution layer: routes, UI, data, radio, commerce, social, safety, and deployment.

`docs/founder/` is the source-of-truth layer: brand doctrine, language rules, product philosophy, care principles, and system logic.

No page, feature, CTA, prompt, campaign, show script, product copy, notification, onboarding step, or moderation surface should contradict these files.

## Hard corrections

HOTMESS does not do affiliate mechanics.

HOTMESS does not do gamification.

That means no points, streaks, XP, quests, leaderboards, ranks, badges-for-behaviour, addictive loops, or behavioural manipulation dressed as community.

If legacy docs or code mention challenges, leaderboards, opt-ins, rewards, or growth loops, treat them as deprecated unless explicitly reframed as non-gamified editorial, care, event, or utility surfaces.

## File groups

- `canon/` — worldview, emotional thesis, masculinity, nightlife, care
- `product-philosophy/` — doctrine for radio, HNH MESS, apparel, records
- `language/` — banned language, CTAs, consent, navigation, microcopy
- `systems/` — homepage logic, crosslinking, ecosystem hierarchy, user state
- `legal/` — practical compliance rails for product and copy

## Use rule

When generating new work, read these in order:

1. `canon/ecosystem-overview.md`
2. `canon/emotional-thesis.md`
3. `language/banned-language.md`
4. `language/consent-language.md`
5. The relevant product or system file

Then ship the work with no dead ends, no fake care, and no addictive mechanics.
