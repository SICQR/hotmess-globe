# Trust System Spec

## Purpose

Trust defines operational credibility.

Trust determines:
- amplification tolerance
- ranking persistence
- moderation confidence
- suppression thresholds
- visibility durability
- economic flexibility

Trust is not cosmetic reputation.
Trust is operational permission.

## Core Principles

- trust is earned
- trust decays
- trust is contextual
- trust is reversible
- trust must resist manipulation
- trust cannot be permanently bought

## Trust Layers

### User Trust
Measures reliability of user behavior.

### Venue Trust
Measures operational reliability.

### Event Trust
Measures event accuracy and timing reliability.

### Promoter Trust
Measures amplification reliability.

### District Trust Context
Measures ecosystem confidence within a geography.

## Trust Bands

- Restricted: 0–24
- Low: 25–49
- Normal: 50–69
- Trusted: 70–84
- High Trust: 85–100

## Trust Inputs

### Positive
- verified identity
- accurate updates
- corroborated claims
- moderation consistency
- low dispute rates
- long-term reliability

### Negative
- spam
- misinformation
- harassment
- impersonation
- location abuse
- synthetic amplification
- unresolved disputes

## Trust Decay

Trust must decay over time.

Decay factors:
- inactivity
- stale updates
- contradictory behavior
- moderation actions
- spam velocity
- abuse recurrence

## Trust Recovery

Recovery requires:
- sustained accuracy
- successful moderation review
- low incident recurrence
- verified corrections
- cooling periods

## Trust Constraints

Trust may influence:
- persistence
- amplification efficiency
- cooldowns
- suppression thresholds

Trust may NEVER:
- override safety
- override truth
- create permanent dominance

## Trust Enforcement

Low trust should trigger:
- faster decay
- stricter throttling
- higher corroboration requirements
- lower boost efficiency
- shorter visibility windows

High trust should trigger:
- longer persistence
- softer decay
- stronger corroboration weighting
- moderate amplification flexibility

## Verification Doctrine

Verification is layered.

Verification tiers:
- unverified
- soft verified
- verified
- high trust
- restricted

Verification must remain revocable.

## Trust Observability

Trust systems must expose:
- trust migration
- trust inflation
- trust collapse
- recovery velocity
- dispute clustering
- moderation reversals

## Immutable Rules

- trust cannot bypass safety
- trust cannot fake freshness
- trust cannot erase moderation history
- trust must remain observable
- trust systems must remain reversible
