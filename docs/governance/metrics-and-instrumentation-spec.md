# Metrics and Instrumentation Spec

## Purpose

Metrics exist to measure:
- truth integrity
- readability
- trust health
- saturation pressure
- moderation performance
- launch readiness
- abuse resistance

Metrics are operational controls.
Not vanity analytics.

## Priority Hierarchy

1. Safety metrics
2. Truth metrics
3. Trust metrics
4. Readability metrics
5. Density metrics
6. Engagement metrics
7. Monetization metrics

## Core Metric Families

### Safety Metrics
Track:
- harassment rates
- stalking-risk suppressions
- unsafe amplification
- repeat offender recurrence
- moderation intervention rates

### Truth Metrics
Track:
- stale leakage
- contradiction rate
- correction latency
- false busy rate
- false empty rate
- verification coverage

### Trust Metrics
Track:
- trust distribution
- trust decay
- trust inflation
- dispute frequency
- recovery rates
- verification reversals

### Readability Metrics
Track:
- cluster overload
- label collision
- visible item count
- abandonment rates
- scan failure behavior
- duplicate suppression

### Density Metrics
Track:
- district congestion
- signals per km²
- source concentration
- category dominance
- saturation pressure

### Ranking Metrics
Track:
- route starts
- save actions
- confirmation usefulness
- rank reversals
- trusted relevance score

### Monetization Metrics
Track:
- paid surface share
- sponsor concentration
- boost saturation
- payment-to-utility ratio

## Health Indicators

Critical indicators:
- signal inflation rate
- stale leakage rate
- correction latency
- trust-weighted engagement
- false urgency rate
- visibility half-life
- district congestion

## District Metrics

Each district must expose:
- trust distribution
- boost concentration
- moderation load
- stale leakage
- density trend
- activity momentum
- category balance

## Instrumentation Rules

Metrics must be:
- segmented
- time-aware
- thresholded
- observable
- alertable
- tied to action ownership

## Observability Requirements

Every critical metric must support:
- alert thresholds
- anomaly detection
- district comparison
- source-type segmentation
- historical trend analysis

## Failure Doctrine

The system is unhealthy when:
- stale visibility rises
- density becomes unreadable
- trust inflates unnaturally
- boosts distort usefulness
- moderation cannot keep pace
- synthetic urgency appears

## Immutable Rules

- quiet states are valid
- density without utility is failure
- truth outranks engagement
- readability outranks monetization
- metrics must trigger action
