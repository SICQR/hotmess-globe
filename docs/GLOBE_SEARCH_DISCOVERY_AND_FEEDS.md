# Globe Search Discovery And Feeds

Purpose: define how discovery, search, feed surfaces, contextual recommendations, and signal exploration behave across the HOTMESS Globe.

This document governs:

- Globe discovery;
- realtime feeds;
- contextual rails;
- search behaviour;
- feed ranking;
- district discovery;
- beacon exploration;
- social intent routing;
- anti-spam visibility;
- privacy-safe recommendations.

The Globe should feel:

```txt
alive, atmospheric, contextual, and human
```

not:

```txt
algorithmically manipulative, hyper-addictive, or popularity-obsessed
```

---

# Core philosophy

Discovery should:

- help people orient;
- help people connect;
- help people explore safely;
- surface atmosphere;
- surface culture;
- surface care;
- surface relevant energy.

Discovery must NOT:

- maximise screen addiction;
- maximise doom scrolling;
- maximise outrage;
- maximise popularity loops;
- manufacture fake urgency;
- become creator-clout infrastructure.

---

# Anti-gamification discovery policy

The Globe does NOT:

- trend people publicly;
- expose popularity ladders;
- rank users by clout;
- expose public interaction counts;
- create virality mechanics.

Forbidden:

```txt
Top users nearby
```

```txt
Most popular profile tonight
```

```txt
Trending hotties in Soho
```

```txt
Your beacon is blowing up
```

---

# Core discovery model

The Globe is:

```txt
contextual discovery infrastructure
```

not:

```txt
an infinite engagement feed
```

The system should prioritise:

- relevance;
- safety;
- freshness;
- consent;
- atmosphere;
- local context;
- user intention.

# Feed philosophy

Feeds should feel:

- finite;
- breathable;
- intentional;
- contextual;
- low-pressure.

Avoid:

- endless scroll addiction;
- infinite refresh loops;
- slot-machine interaction patterns;
- ragebait ranking.

---

# Feed structure

Recommended feed structure:

```txt
Now
Tonight
Nearby
Selected
Saved
Care
Radio
Events
Market
```

Feeds should:

- degrade gracefully;
- stay location-aware;
- remain privacy-safe;
- avoid repetitive duplication.

---

# Feed freshness rules

Feeds must remain honest.

Good:

```txt
Recently active nearby
```

Bad:

```txt
Live now
```

without freshness proof.

Stale content should:

- decay naturally;
- fade visually;
- reduce prominence;
- eventually expire.

---

# Search philosophy

- support intentional exploration;
- support safe discovery;
- reduce overwhelm;
- avoid manipulation.

Search must NOT:

- expose exact hidden location;
- expose private attendance;
- expose SOS/help users;
- expose hidden recovery participation.

# Intent-aware discovery

Discovery should adapt to:

- user intent;
- time of day;
- local energy;
- saved preferences;
- visibility policy;
- accessibility preferences;
- reduced stimulation mode.

Examples:

A user in:

```txt
low stimulation mode
```

should NOT receive:

- hyper-dense nightlife rails;
- constant social prompts;
- rapid pulse surfaces.

---

# Local district discovery

Districts are atmospheric surfaces.

District view may show:

- warmth;
- activity level;
- active signals;
- venue glow;
- live events;
- radio presence;
- care availability.

Districts must NOT expose:

- exact user density;
- precise crowd movement;
- hidden attendance;
- identifiable recovery participation.

---

# Density-aware discovery

Dense areas require aggregation.

Instead of:

```txt
312 overlapping profiles
```

use:

```txt
Soho is active tonight
```

or:

```txt
Several nearby Chill signals
```

This preserves:

- readability;
- privacy;
- emotional calm.

# Reduced stimulation discovery

Reduced stimulation mode should:

- reduce pulse intensity;
- reduce animation;
- reduce density overlays;
- simplify feed hierarchy;
- suppress nightlife overload.

Discovery should feel:

```txt
quieter and more breathable
```

---

# Search result presentation

Search results should:

- feel curated;
- remain finite;
- avoid overload;
- preserve readability.

Preferred:

```txt
12 relevant nearby signals
```

Avoid:

```txt
3,948 results
```

---

# Feed interruption policy

Feeds should not constantly reorder aggressively.

Avoid:

- infinite reshuffling;
- jumpy ranking;
- attention hijacking;
- engagement baiting.

Prefer:

- stable ordering;
- calm transitions;
- contextual updates.

---

# Sponsored discovery policy

Sponsored discovery:

- must be labelled;
- must remain sparse;
- must never imitate care or safety;
- must never dominate local discovery.

Forbidden:

- fake community signals;
- fake popularity;
- fake local warmth.

---

# Accessibility

Discovery systems must support:

- reduced motion;
- reduced stimulation;
- screen readers;
- keyboard navigation;
- high contrast;
- low-cognitive-load layouts.

Feeds should remain understandable without:

- animation;
- colour-only signalling;
- dense realtime transitions.

# Acceptance criteria

The discovery system succeeds when:

- users discover relevant energy without overwhelm;
- discovery feels atmospheric rather than manipulative;
- local context remains readable at scale;
- exact location never leaks;
- care remains visible without exposing vulnerability;
- feeds remain finite and breathable;
- search supports intention rather than addiction;
- discovery feels culturally alive;
- sponsored content never dominates;
- popularity does not become the organising principle of the Globe.
