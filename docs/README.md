# HOTMESS Docs

## Purpose

This directory contains the canonical governance, ranking, trust, moderation, launch, and operational doctrine for HOTMESS.

HOTMESS is a governed realtime visibility system.

The platform prioritizes:
- truth
- trust
- safety
- freshness
- readability
- local usefulness

Above:
- engagement
- infinite amplification
- synthetic activity

# System Doctrine

The system should:
- reward trustworthy realtime participation
- discourage manipulation
- suppress spam amplification
- surface local signal quickly
- decay stale activity aggressively
- preserve readability under load

The system should never:
- fabricate activity
- fake density
- optimize rage loops
- privilege monetization over trust
- reward spam velocity

---

# Engineering Requirement

Every policy must be implementable as:
- scoring
- thresholds
- state transitions
- moderation actions
- caps
- visibility rules
- instrumentation
- alerts

If it cannot be enforced operationally, it is incomplete.
