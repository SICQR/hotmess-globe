# Observability and Alerting Spec

## Purpose

Define how HOTMESS:
- detects problems
- escalates incidents
- routes ownership
- protects truth and readability

Observability exists to preserve:
- safety
- truth
- trust
- readability
- launch stability

---

# Alert Priorities

1. Safety
2. Truth
3. Trust
4. Readability
5. Saturation
6. Launch Stability
7. Monetization Side Effects

---

# Alert Classes

## Info
Expected system transitions.

## Watch
Concerning trends.

## Warning
Likely future damage.

## Critical
Active system damage.

## Emergency
Immediate intervention required.

---

# Trigger Types

- threshold breach
- rate-of-change spike
- anomaly detection
- policy violation
- source concentration anomaly
- stale leakage spike
- trust collapse

# Core Domains

## Safety Observability
Track:
- harassment
- stalking-risk events
- unsafe visibility
- moderator emergency actions

## Truth Observability
Track:
- contradiction rate
- stale leakage
- correction latency
- false urgency

## Trust Observability
Track:
- trust inflation
- trust collapse
- verification reversals
- dispute clustering

## Readability Observability
Track:
- clutter score
- cluster overload
- abandonment patterns
- label collisions

## Saturation Observability
Track:
- promoter dominance
- paid visibility concentration
- repeated source flooding

# Containment Actions

Automatic containment may:
- freeze sources
- reduce boost reach
- tighten district caps
- increase cooldowns
- collapse clusters sooner
- require verification

Containment must remain proportional.

---

# Incident Lifecycle

1. Observed
2. Triaged
3. Contained
4. Investigating
5. Mitigated
6. Resolved
7. Monitored

---

# Alert Routing

Safety alerts:
- moderation first

Truth alerts:
- ranking plus moderation

Trust alerts:
- trust systems owner

Saturation alerts:
- ops plus ranking

---

# Observability Invariants

- safety alerts outrank all others
- quiet states are not incidents
- alert fatigue is system failure
- every critical alert requires owner
- containment must be reversible
