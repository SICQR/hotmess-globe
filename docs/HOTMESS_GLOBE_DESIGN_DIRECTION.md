# HOTMESS Globe — Design Direction

## 1. Creative North Star

The globe is not a map.

It is the **HOTMESS signal body**: a living, dark, cinematic network showing where energy, care, radio, community, and movement are pulsing.

Design phrase:

```txt
NASA-grade signal intelligence meets queer underground nightlife infrastructure.
```

The experience should feel:

- Premium
- Nocturnal
- Controlled
- Hyper-detailed on approach
- Sparse and mythic from distance
- Masculine, warm, care-first
- Alive without becoming noisy

## 3. Visual System

### Palette

| Role | Colour | Use |
|---|---:|---|
| Void black | `#000000` | Space, background, premium silence |
| Deep app black | `#050507` | Panels, vignettes, overlays |
| HOTMESS gold | `#C8962C` | Core hubs, arcs, active city signal |
| Signal yellow | `#FFEB3B` | Legacy/high-energy beacon highlights |
| Care white | `#FFFFFF` | Recovery, aftercare, safety nodes |
| Radio cyan | `#00C2E0` | Live/audio/listener energy |
| Muted text | `#8A8A8A` | Labels, secondary metadata |

Design rule:

```txt
Gold leads. White protects. Cyan signals live audio. Everything else stays quiet.
```

---

## 4. Node Hierarchy

### Core hub

Example: London.

Visual:

- Largest gold point
- Soft gold glow
- Slow radar ring
- Always visible
- Label can be persistent

Meaning:

```txt
Origin signal / HOTMESS heart.
```

### City node

Examples: Berlin, Amsterdam, Paris, Barcelona, Ibiza, NYC, Tokyo.

Visual:

- Medium gold point
- Label appears at low/mid zoom
- Optional faint pulse
- Connected by signal arcs

Meaning:

```txt
A recognised HOTMESS signal city.
```

### Live signal

Visual:

- Small cyan or white micro-dot
- Only appears mid/high zoom
- Can pulse gently

Meaning:

```txt
Something is happening now.
```

### Care / recovery node

Visual:

- White dot or ring
- Strong tap target
- Always visible above noise
- Never styled like nightlife bait

Meaning:

```txt
Safe landing point.
```

### Archive / passive beacon

Visual:

- Dim amber point
- No ring
- High zoom only

Meaning:

```txt
Historical or inactive signal.
```

## 5. Density Behaviour

### Far view

Design intent:

```txt
Mythic planet. Big signals only.
```

Visible:

- Core hubs
- Major city nodes
- Major arcs
- Soft regional glow

Hidden:

- Individual low-energy beacons
- Tiny labels
- Passive/archive nodes

### Mid view

Design intent:

```txt
Network comes into focus.
```

Visible:

- City labels
- Signal clusters
- Care anchors
- Live nodes above threshold

### Close view

Design intent:

```txt
Hyper-detail without chaos.
```

Visible:

- Individual beacons
- Venue/service/affiliate detail
- Better labels
- Clickable cards

## 6. Motion Direction

Motion should feel expensive.

Use:

- Slow pulse
- Slow ring propagation
- Very restrained arcs
- Soft breathing glows
- No frantic bouncing
- No arcade neon

Do not use:

- Fast spin
- Excessive flashing
- Random particle storms
- Rainbow gradients
- Constant marker popping

Motion principle:

```txt
The globe is breathing, not shouting.
```

---

## 7. UI Overlay Direction

The globe should own the screen.

UI should sit like instrumentation, not like a SaaS dashboard.

### Top layer

- Page title: `PULSE`
- Short status line: `LIVE SIGNALS FROM THE HOTMESS NETWORK`
- Optional city count / active signals count

### Bottom nav relationship

The globe should cross-link to:

- Radio
- Care
- Shop
- Affiliate
- Community

### Node card

When tapping a node:

```txt
[City / Node name]
Type
Live status
Primary CTA
Secondary care/action CTA
```

Example:

```txt
LONDON
CORE HUB
HOTMESS signal is active

Listen Live
Find Care
```

---

## 8. Typography

Direction:

- Uppercase for system labels
- Tight tracking for instrumentation
- Strong weight for node names
- Short copy only

Example labels:

```txt
CORE HUB
LIVE SIGNAL
CARE NODE
CITY NODE
RADIO RELAY
```

Avoid:

- Paragraph-heavy cards
- Cute tone
- Explanatory UI bloat

## 10. Design Acceptance Criteria

A pass means:

- At first glance, the globe feels premium and cinematic.
- At low zoom, it is not cluttered.
- At high zoom, it feels rich and detailed.
- Gold is the dominant signal language.
- Care/recovery nodes are visible and calm.
- Live/radio energy is distinct but not overpowering.
- No node type is visually ambiguous.
- No interaction creates a dead end.

A fail means:

- It looks like Google Maps with glitter.
- It becomes neon/rainbow chaos.
- Every dot looks equally important.
- Labels overwhelm the globe.
- Care nodes disappear into nightlife styling.

---

## 11. Implementation Order From A Design Perspective

1. **Density hierarchy** — make the globe readable at every zoom.
2. **Node taxonomy** — each beacon type gets a visual role.
3. **Signal arcs** — make the network feel connected.
4. **Core hub treatment** — London becomes the signal heart.
5. **Care node styling** — visible, calm, trusted.
6. **Node cards** — every tap has a clear next action.
7. **Live energy polish** — radio/listener pulses and right-now motion.

---

## 12. One-line design brief

```txt
Design the globe like a private global signal system for queer nightlife, radio, care, and community — cinematic from space, precise on approach.
```
